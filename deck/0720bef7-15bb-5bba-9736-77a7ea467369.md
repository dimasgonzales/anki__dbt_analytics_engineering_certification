<front>

How do you define model-specific configurations, such as materialization types or indexes, directly within a SQL model file?

</front>

---

<back>

Use the `{{ config() }}` Jinja macro at the top of the file. This method allows you to set properties like `materialized`, `schema`, or database-specific options, overriding project-level defaults.

Example:
```sql
{{ config(materialized='table') }}
```

</back>
