<front>

Why are database sequences (integer incrementing keys) generally discouraged for dbt models configured as `views`?

</front>

---

<back>

Database sequences are stateful and increment every time a value is requested. Since a view runs its query every time it is accessed, the IDs would change on every select statement, breaking referential integrity and joining logic.

</back>
