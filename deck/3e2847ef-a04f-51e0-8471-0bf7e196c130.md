<front>

Which `on_schema_change` settings are valid when using enforced contracts with incremental models?

</front>

---

<back>

You must use either `append_new_columns` or `fail`.

The default setting (`ignore`) is not allowed because it permits the underlying table schema to drift from the defined YAML contract.

</back>
