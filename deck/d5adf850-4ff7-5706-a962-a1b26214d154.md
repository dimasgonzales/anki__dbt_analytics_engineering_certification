<front>

You are organizing your YAML files within subdirectories. Why is it considered a best practice to prefix these filenames with an underscore (e.g., `_int_finance__models.yml`)?

</front>

---

<back>

The underscore ensures the configuration file sorts to the **top of the file explorer** within that directory. This makes it easy to locate the documentation and config file relative to the model SQL files.

</back>
