<front>

What Jinja syntax is used within a `.yml` file to reference a description defined in a separate Markdown doc block?

</front>

---

<back>

Use the `doc` function.

Syntax:
```yaml
description: '{{ doc("doc_block_name") }}'
```
The doc block must be defined in a `.md` file within your project using `{% docs doc_block_name %}`.

</back>
