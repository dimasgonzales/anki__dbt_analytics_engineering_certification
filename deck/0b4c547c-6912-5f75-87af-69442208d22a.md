<front>

What file naming convention must be used for SQL files associated with versioned models in dbt?

</front>

---

<back>

Files must follow the pattern `<model_name>_v<version>.sql`.

Example:
For a model named `dim_customers` version 2, the file must be named `dim_customers_v2.sql`.

</back>
