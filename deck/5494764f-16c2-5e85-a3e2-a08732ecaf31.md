<front>

You are orchestrating dbt Core with Airflow and want to ensure that dbt's Python dependencies do not conflict with the Airflow worker's environment. Which Airflow Operator should you use?

</front>

---

<back>

You should use the `KubernetesPodOperator` (or `DockerOperator`). 

Unlike the `BashOperator`, which runs commands directly on the worker, the `KubernetesPodOperator` executes dbt in an isolated container, ensuring complete separation of dependencies and resources.

</back>
