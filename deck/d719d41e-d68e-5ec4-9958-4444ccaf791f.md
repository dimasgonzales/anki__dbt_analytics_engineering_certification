<front>

You notice that `package-lock.yml` is currently listed in your project's `.gitignore` file. Why is this considered a bad practice for production deployments?

</front>

---

<back>

Excluding the lock file means the production environment will resolve dependencies dynamically when `dbt deps` runs. This could install newer, untested versions of packages that differ from Development, potentially causing breaking changes or unexpected behavior.

</back>
