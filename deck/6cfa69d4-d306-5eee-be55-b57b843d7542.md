<front>

How do you configure a specific SQL model's materialization strategy directly within the model file?

</front>

---

<back>

Use the Jinja `config()` macro at the top of the SQL file. This allows you to set properties like `materialized`, `schema`, or `tags` for that specific model.

Example:
```sql
{{ config(
    materialized='table'
) }}
```

</back>
