<front>

How does dbt determine column data types for seeds by default, and how can you override this?

</front>

---

<back>

By default, dbt infers datatypes by scanning the CSV data. To override this inference (e.g., to preserve leading zeros), you must explicitly define types using the `+column_types` configuration.

</back>
