<front>

You are deploying a Materialized View in dbt, but you want to ensure that if the model logic changes, dbt automatically updates the existing database object to match. What configuration is required?

</front>

---

<back>

You must configure `on_configuration_change='apply'`.

This setting tells dbt to attempt to alter the existing object to match the new configuration. If set to `continue`, dbt will warn but leave the object as-is.

```yaml
config(
  materialized = 'materialized_view',
  on_configuration_change = 'apply'
)
```

</back>
