<front>

What action does dbt automatically take if the target schema defined in `profiles.yml` does not exist in the database during execution?

</front>

---

<back>

dbt checks for the schema's existence and automatically runs a `CREATE SCHEMA IF NOT EXISTS` (or database equivalent) command. Users typically do not need to manually create schemas before running dbt.

</back>
