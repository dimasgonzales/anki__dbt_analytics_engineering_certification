<front>

What are the two snapshot strategies available in dbt and when should each be used?

</front>

---

<back>

dbt supports **timestamp** and **check** strategies.

*   **timestamp**: The best practice. Use when the source has a reliable `updated_at` column. It handles schema changes automatically.
*   **check**: The fallback. Use when no timestamp exists. It compares values in specific columns (`check_cols`) to detect changes.

</back>
