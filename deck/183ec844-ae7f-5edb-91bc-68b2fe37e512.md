<front>

You want to configure a model to use a different warehouse size in Production vs. Development. Instead of using complex Jinja logic like `{% if target.name == 'prod' %}`, what is the best practice approach?

</front>

---

<back>

Use an **Environment Variable**.

Set a variable (e.g., `DBT_WAREHOUSE_SIZE`) in the dbt Cloud Environment settings and reference it in your config:
```yaml
+snowflake_warehouse: "{{ env_var('DBT_WAREHOUSE_SIZE', 'x-small') }}"
```
This approach avoids parsing issues and keeps logic out of the codebase.

</back>
