<front>

You need to rename the `user_id` column to `customer_uuid` in a staging model. How can you verify if this change will break a specific downstream dashboard?

</front>

---

<back>

Use dbt's Column-Level Lineage features (such as in dbt Explorer) to trace the `user_id` column forward. This reveals exactly which Exposures (dashboards) rely on that specific field, whereas table lineage would only show that the dashboard relies on the model generally.

</back>
