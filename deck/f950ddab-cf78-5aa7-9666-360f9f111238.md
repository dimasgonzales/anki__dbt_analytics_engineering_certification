<front>

What is the primary function of the `dbt parse` command, and how does it differ from `dbt compile` regarding database connectivity?

</front>

---

<back>

`dbt parse` validates Jinja/YAML syntax and generates the `manifest.json` file representing project resources. Unlike `dbt compile`, `dbt parse` **does not** connect to the data warehouse, meaning it cannot generate fully compiled SQL that relies on database introspection.

</back>
