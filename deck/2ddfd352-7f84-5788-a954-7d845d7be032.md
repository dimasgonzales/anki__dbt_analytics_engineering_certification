<front>

Given the macro definition `{% macro calculate_tax(region, rate) %}`, what happens if you run `dbt run-operation calculate_tax --args '{zone: "US", rate: 0.05}'`?

</front>

---

<back>

The macro will likely fail (or the argument will be undefined) because the key `zone` in the arguments does not match the expected argument `region` in the macro definition.

dbt requires the keys in the `--args` dictionary to match the macro's argument names exactly.

</back>
