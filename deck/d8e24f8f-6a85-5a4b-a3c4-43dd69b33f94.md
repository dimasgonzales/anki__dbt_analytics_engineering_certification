<front>

When adding documentation for a custom generic test in a `schema.yml` file under the `macros:` key, what naming convention must be used?

</front>

---

<back>

You must prepend `test_` to the generic test's name.

For example, if the test is defined as `{% test is_even ... %}`, the documentation block must reference it as `name: test_is_even`.

</back>
