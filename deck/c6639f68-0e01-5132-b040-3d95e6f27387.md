<front>

Your incremental model uses the `merge` strategy, but you only want to update the `status` column when a duplicate `id` is found, leaving other columns unchanged. How do you configure this?

</front>

---

<back>

Use the `merge_update_columns` configuration to whitelist specific columns for updates.

```yaml
{{ config(
    materialized = 'incremental',
    unique_key = 'id',
    incremental_strategy = 'merge',
    merge_update_columns = ['status']
) }}
```

</back>
