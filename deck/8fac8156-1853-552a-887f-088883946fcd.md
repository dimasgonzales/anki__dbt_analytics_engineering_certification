<front>

Your CI pipeline times out during the documentation step because your warehouse is too large. How can you configure dbt to only generate documentation for models changed in the PR?

</front>

---

<back>

Use `dbt docs generate` with the `--select` flag and state comparison.

```bash
dbt docs generate --select state:modified --state path/to/artifacts
```

</back>
