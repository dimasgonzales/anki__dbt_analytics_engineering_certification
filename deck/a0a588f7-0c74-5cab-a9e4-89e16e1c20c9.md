<front>

You have `+materialized: view` set at the project root level in `dbt_project.yml` and `+materialized: table` set for the `marketing` subdirectory. How will a model inside the `marketing` folder be materialized?

</front>

---

<back>

It will be materialized as a **table**. Configurations applied to specific subdirectories (scopes) override general configurations set at higher levels in the directory tree.

</back>
