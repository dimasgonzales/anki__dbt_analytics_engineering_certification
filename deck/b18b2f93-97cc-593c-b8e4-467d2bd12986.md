<front>

Your production pipeline runs `dbt run --warn-error`. After upgrading dbt versions, the pipeline fails immediately despite no changes to the model code. What is the most likely cause?

</front>

---

<back>

The dbt upgrade likely introduced a new **deprecation warning**. Because `--warn-error` is enabled, this otherwise benign warning was promoted to an error, causing the pipeline to fail. This is why using granular warning options is often safer for production than the global switch.

</back>
