<front>

You are setting up a CI pipeline that includes the `dbt_project_evaluator` audit package. Why is it best practice to run the project build first, excluding the audit package, before running the audit package separately?

</front>

---

<back>

It ensures that the project models are valid and built successfully before auditing them. It also separates build errors from audit standard violations.

Workflow:
1. `dbt build --select state:modified+ --exclude package:dbt_project_evaluator`
2. `dbt build --select package:dbt_project_evaluator`

</back>
