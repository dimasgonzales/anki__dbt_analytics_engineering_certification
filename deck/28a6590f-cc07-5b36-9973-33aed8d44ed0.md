<front>

What feature in dbt allows you to execute SQL statements, such as `VACUUM` or `GRANT`, immediately after a model is built?

</front>

---

<back>

`post-hooks`. These are SQL statements defined in the model configuration that run after the model is built. They are often used for database maintenance tasks, indexing, or granting permissions not covered by standard dbt functionality.

</back>
