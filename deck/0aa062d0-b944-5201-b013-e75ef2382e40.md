<front>

What is the primary purpose of using macros from the `dbt` namespace (e.g., `dbt.concat`, `dbt.hash`) instead of raw SQL functions?

</front>

---

<back>

These macros abstract SQL syntax differences to ensure **cross-database compatibility**. By using them, your project code remains portable and compiles to the correct dialect-specific SQL (e.g., Snowflake, BigQuery, Postgres) without needing manual refactoring if the underlying adapter changes.

</back>
