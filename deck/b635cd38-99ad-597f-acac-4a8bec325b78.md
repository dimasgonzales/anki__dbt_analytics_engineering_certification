<front>

A developer has made several experimental changes in the dbt Cloud IDE that have resulted in a messy state, and they wish to discard all work and revert to the version currently on GitHub/GitLab. How should they proceed?

</front>

---

<back>

They should use the **Rollback to remote** feature found under the git actions menu (often an ellipsis `...`) in the IDE. This effectively performs a `git reset --hard` to the remote branch state.

</back>
