<front>

What is the primary purpose of the `--defer` flag (or 'Defer to production' setting) in dbt?

</front>

---

<back>

It allows developers to run specific models without building their upstream parents locally. By referencing a **state** file (manifest.json) from another environment (like production), dbt treats upstream dependencies as existing tables in that environment rather than requiring a local rebuild.

</back>
