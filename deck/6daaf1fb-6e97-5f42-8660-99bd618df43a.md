<front>

You are reviewing a production issue where a dashboard broke after a manual table drop-and-replace of a static lookup table. How does using `dbt seed` prevent this specific issue?

</front>

---

<back>

Because `dbt seed` updates data by truncating the table content rather than dropping the table object itself, the view definitions relying on that table remain valid and intact in the data warehouse.

</back>
