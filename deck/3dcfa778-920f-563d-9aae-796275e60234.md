<front>

When using the `check` strategy in a dbt snapshot, which configuration argument is mandatory to identify which changes trigger a new row version?

</front>

---

<back>

The `check_cols` argument. It requires a list of column names to monitor for changes, or the string `'all'` to check every column.

Example:
```yaml
strategy: check
check_cols: ['status', 'amount']
```

</back>
