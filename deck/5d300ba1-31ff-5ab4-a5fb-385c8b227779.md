<front>

Why is it considered a best practice to use `target.name` checks within the `generate_schema_name` macro?

</front>

---

<back>

It allows for environment-aware behavior. You can configure dbt to use clean schema names (e.g., `marketing`) in production while keeping prefixed, isolated schemas (e.g., `dbt_alice_marketing`) in development to prevent collisions.

</back>
