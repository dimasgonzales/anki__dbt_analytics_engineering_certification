<front>

How does dbt Cloud prevent schema collisions when multiple Pull Requests trigger CI jobs simultaneously?

</front>

---

<back>

dbt Cloud creates a unique, temporary schema for each Pull Request run.

These schemas usually follow a naming convention like `dbt_cloud_pr_<job_id>_<pr_id>`, ensuring complete isolation between concurrent CI jobs.

</back>
