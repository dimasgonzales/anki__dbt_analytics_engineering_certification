<front>

A developer is spending significant time adding `{%-` and `-%}` tags to their macros because the compiled SQL has too many blank lines. What advice should you give them?

</front>

---

<back>

Advise them to stop optimizing for compiled whitespace. 

Unless the whitespace causes a functional syntax error in the database (which is rare), it does not impact performance. They should focus on keeping the Jinja source code clean and readable for future maintainers.

</back>
