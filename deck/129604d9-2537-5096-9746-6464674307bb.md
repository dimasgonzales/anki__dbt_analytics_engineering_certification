<front>

How do you reference an environment variable in `profiles.yml` or `dbt_project.yml`, and how do you set a fallback value?

</front>

---

<back>

Use the Jinja `env_var` function. The first argument is the variable name, and the optional second argument is the default value used if the variable is not present.

```yaml
target: "{{ env_var('DBT_TARGET', 'dev') }}"
```

</back>
