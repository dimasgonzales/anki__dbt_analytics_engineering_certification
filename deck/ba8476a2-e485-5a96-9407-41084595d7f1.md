<front>

Your `dbt_project.yml` configures all models in the `marts` directory to be views, but you need one specific model in that directory to be a table. How do you achieve this?

</front>

---

<back>

Add a config block to the top of that specific model's SQL file setting `materialized='table'`. In-file configurations provided via the `config()` macro take precedence over configurations defined in `dbt_project.yml`.

</back>
