<front>

You are utilizing `dbt_utils.get_column_values` to pivot a column dynamically. Why does dbt wait to render this model until runtime instead of during the initial parsing phase?

</front>

---

<back>

The macro is introspective, meaning it queries the database. dbt uses Just-In-Time (JIT) rendering to ensure the SQL is generated using the most up-to-date database state, which may include tables created earlier in the current run.

</back>
