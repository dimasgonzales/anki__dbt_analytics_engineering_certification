<front>

What is the primary difference between the static documentation generated by `dbt docs generate` and dbt Explorer regarding data freshness?

</front>

---

<back>

Static documentation is a snapshot in time representing the project structure when generated. **dbt Explorer** provides stateful awareness, integrating **execution metadata** from the Production environment to display the real-time pass/fail status of models and source freshness alongside documentation.

</back>
