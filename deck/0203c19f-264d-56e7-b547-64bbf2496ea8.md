<front>

You are processing a high-volume, date-partitioned dataset on BigQuery and need to re-process data for specific past dates without scanning the entire history. Which configuration is best?

</front>

---

<back>

Use `insert_overwrite` combined with `partition_by`. This allows dbt to drop and replace only the partitions included in your current run's source data.

```yaml
{{ config(
    materialized = 'incremental',
    incremental_strategy = 'insert_overwrite',
    partition_by = {'field': 'date_col', 'data_type': 'date'}
) }}
```

</back>
