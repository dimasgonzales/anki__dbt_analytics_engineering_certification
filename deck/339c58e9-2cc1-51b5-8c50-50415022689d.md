<front>

How can you replace the logic of a core dbt test (like `unique` or `not_null`) for your entire project without modifying every YAML file?

</front>

---

<back>

Create a generic test file (e.g., `tests/generic/unique.sql`) and define a test with the exact same name (`unique`).

dbt prioritizes project-defined tests over the global internal implementations, effectively overriding the default behavior.

</back>
