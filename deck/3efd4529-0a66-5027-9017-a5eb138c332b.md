<front>

A model is failing with a SQL syntax error during `dbt run`. How would you efficiently identify and debug the exact SQL causing the issue?

</front>

---

<back>

Locate the specific file in the **`target/run/`** directory corresponding to the failing model. 

This file contains the fully compiled DDL wrapped in the create statement. You can copy this code and paste it directly into your database IDE to reproduce and fix the syntax error.

</back>
