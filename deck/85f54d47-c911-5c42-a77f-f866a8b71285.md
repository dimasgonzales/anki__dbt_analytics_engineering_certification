<front>

Given the following source configuration, will the `orders` table identifier be quoted in the compiled SQL?

```yaml
sources:
  - name: jaffle_shop
    quoting:
      identifier: true
    tables:
      - name: orders
        quoting:
          identifier: false
```

</front>

---

<back>

No, the `orders` table identifier will **not** be quoted.

Although the source `jaffle_shop` defaults to `identifier: true`, the `orders` table explicitly overrides this setting with `identifier: false`.

</back>
