<front>

Your incremental model runs successfully but occasionally updates the wrong rows or results in non-deterministic data in the target table. What troubleshooting step should you take regarding the `unique_key`?

</front>

---

<back>

You should manually add a `unique` test to the columns used in your `unique_key` config on the source or model.

```yaml
columns:
  - name: my_id
    tests:
      - unique
```
If the source data has duplicates on the key, the SQL `MERGE` statement produces undefined behavior.

</back>
