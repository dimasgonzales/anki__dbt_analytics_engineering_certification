<front>

You have an incremental model containing a `where` clause wrapped in `{% if is_incremental() %}`. What happens during the **first** time you run this model?

</front>

---

<back>

The `is_incremental()` macro returns `false` because the destination table does not exist yet. Consequently, the conditional `where` clause is ignored, and dbt builds the table from scratch using all available source data.

</back>
