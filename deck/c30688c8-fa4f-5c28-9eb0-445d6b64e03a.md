<front>

What is the `incremental_predicates` configuration used for in dbt?

</front>

---

<back>

It is a configuration key used with the `merge` incremental strategy to apply filters to the destination table (often aliased as `dbt_internal_target`) during the merge operation. This reduces the number of rows scanned in the target table, significantly improving performance and reducing costs on large datasets.

</back>
