<front>

You need to implement continuous monitoring of data quality failures in real-time. How should you execute your tests in dbt with Materialize?

</front>

---

<back>

Run `dbt test --store-failures`.

In Materialize, this creates a **materialized view** for the test failures. Unlike static tables in other adapters, this view continuously updates, allowing you to alert on failing records as they arrive in the stream.

</back>
