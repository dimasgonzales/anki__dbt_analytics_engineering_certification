<front>

You are configuring `profiles.yml` for a production deployment and need to ensure the database password is not exposed in the codebase. How should you configure the password field?

</front>

---

<back>

You should use `env_var()` to reference an environment variable set in the deployment environment. This keeps the secret out of version control.

Example:
```yaml
pass: "{{ env_var('DBT_PROD_PASSWORD') }}"
```

</back>
