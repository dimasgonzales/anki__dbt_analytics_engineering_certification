<front>

A developer successfully runs a job in a Continuous Integration (CI) environment that adds a new model, but the model does not appear in the project's Explorer view. Why?

</front>

---

<back>

CI jobs are designed to be ephemeral and do **not** update the dbt Catalog. This prevents temporary changes in pull requests from overwriting the production view of the project metadata.

</back>
