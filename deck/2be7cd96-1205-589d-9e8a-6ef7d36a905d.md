<front>

When using `incremental_strategy='merge'`, what configuration is mandatory to identify which records should be updated versus inserted?

</front>

---

<back>

You must define a `unique_key`. dbt uses this key to match records between the source query and the destination table to determine if a row exists (update) or is new (insert).

</back>
