<front>

You have a CI job running `dbt run --select state:modified+` with strict error handling (`error: all`). The job fails when no models have been modified. How do you fix this without disabling strict error handling for everything else?

</front>

---

<back>

You should configure `warn_error_options` to specifically **silence** or **warn** on the `NoNodesForSelectionCriteria` class.

Example CLI:
```bash
dbt run --warn-error-options '{"error": "all", "silence": ["NoNodesForSelectionCriteria"]}'
```

</back>
