<front>

In databases that support sharding like BigQuery, how can you configure a single source definition to match multiple date-sharded tables?

</front>

---

<back>

You can use a wildcard (`*`) in the `identifier` field (e.g., `identifier: "events_*"`). This treats the collection of sharded tables as a single source object within dbt.

</back>
