<front>

How can you configure a dbt model property, such as `target_lag` for a Dynamic Table, to change values based on the environment (e.g., Dev vs. Prod)?

</front>

---

<back>

Use **Jinja conditionals** referencing `target.name` inside the config block or within a helper macro.

This allows you to set aggressive refresh schedules for production and cost-saving schedules for development without changing the source code.

</back>
