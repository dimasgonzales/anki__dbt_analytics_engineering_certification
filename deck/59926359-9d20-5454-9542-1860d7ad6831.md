<front>

You are building a semantic model on top of a pre-aggregated table `daily_sales_summary` that lacks a unique ID column. How do you configure the `semantic_model` block to avoid compilation errors?

</front>

---

<back>

You must define a `primary_entity` even without a matching physical column. This creates a virtual entity for scoping dimensions.

```yaml
semantic_model:
  name: daily_sales_summary
  primary_entity: sales_summary_id # Virtual entity name
```

</back>
