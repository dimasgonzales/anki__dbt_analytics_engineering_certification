<front>

You try to configure a unique test inside a model file using `{{ config(data_tests=['unique']) }}` but receive an error. What is the correct way to apply this test?

</front>

---

<back>

You must define the test in a `.yml` file.

`data_tests` (or `tests`) are **properties**, not configurations, and cannot be defined in a `config()` block. 

Correct approach in `schema.yml`:
```yaml
models:
  - name: my_model
    data_tests:
      - unique
```

</back>
