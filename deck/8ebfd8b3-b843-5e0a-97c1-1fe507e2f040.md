<front>

How does the 'Custom Branch' setting function differently in a Development environment compared to a Deployment environment?

</front>

---

<back>

In a **Development** environment, it sets the 'base branch' that developers check out from and target for PRs. In a **Deployment** environment, it strictly defines which specific git branch the job runner clones and executes.

</back>
