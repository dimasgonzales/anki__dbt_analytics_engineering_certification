<front>

How do you configure a composite `unique_key` (multiple columns) for an incremental model in dbt?

</front>

---

<back>

You define the `unique_key` as a list of strings in the model configuration block.

Example:
```yaml
{{ config(
    materialized='incremental',
    unique_key=['order_id', 'region_id']
) }}
```

</back>
