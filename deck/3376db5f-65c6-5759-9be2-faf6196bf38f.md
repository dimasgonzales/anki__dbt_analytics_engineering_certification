<front>

How does dbt determine precedence between the `store_failures` configuration in YAML files and the `--store-failures` CLI flag?

</front>

---

<back>

The **configuration in the project files** takes precedence. Explicitly setting `store_failures` to `true` or `false` in YAML or SQL overrides the CLI flag. The CLI flag is only used if the config is omitted or set to `none`.

</back>
