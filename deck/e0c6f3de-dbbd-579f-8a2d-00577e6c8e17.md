<front>

You are configuring a production pipeline where data freshness is critical, but documentation updates are secondary. You want to ensure the pipeline finishes even if the documentation step encounters an error. How should you configure the job?

</front>

---

<back>

You should check the **Generate docs on run** option in the Job Settings and ensure `dbt docs generate` is NOT listed in the commands. This ensures that a failure in the documentation step does not mark the run as failed or prevent notifications of success.

</back>
