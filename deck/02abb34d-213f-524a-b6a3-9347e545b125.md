<front>

What is the recommended strategy for organizing model configuration (`.yml`) files in a dbt project?

</front>

---

<back>

Create one configuration file per directory (e.g., `_staging__models.yml`). This creates a balance between centralization (not one file per model) and discoverability (not one giant file for the whole project).

</back>
