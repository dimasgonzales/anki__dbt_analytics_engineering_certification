<front>

You have a column `order_status` that must only contain the values 'placed', 'shipped', or 'returned'. How do you enforce this in dbt?

</front>

---

<back>

Use the `accepted_values` test and provide the list of valid strings in the `values` argument.

Example:
```yaml
columns:
  - name: order_status
    tests:
      - accepted_values:
          values: ['placed', 'shipped', 'returned']
```

</back>
