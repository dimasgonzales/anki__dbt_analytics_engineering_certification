<front>

You are writing an incremental model using the `microbatch` strategy. How do you ensure the SQL processes only the data relevant to the current batch window?

</front>

---

<back>

You must filter your source data using the event time boundaries provided by `model.batch`.

Example:
```sql
where event_timestamp >= '{{ model.batch.event_time_start }}'
  and event_timestamp < '{{ model.batch.event_time_end }}'
```

</back>
