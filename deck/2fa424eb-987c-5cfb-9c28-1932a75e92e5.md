<front>

A junior engineer proposes keeping a container running `dbt docs serve` in production to host documentation for the business. Why is this considered bad practice?

</front>

---

<back>

Running `dbt docs serve` in production is insecure and inefficient. It is a lightweight development server not intended for public traffic or secure access control.

**Best Practice:** Deploy the static files (`index.html`, `.json`) to a secure static hosting provider (e.g., Netlify, S3, GitHub Pages) instead.

</back>
