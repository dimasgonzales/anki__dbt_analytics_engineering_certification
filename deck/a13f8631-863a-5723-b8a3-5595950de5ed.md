<front>

A `not_null` test is failing in production, but you need to see exactly which records contain null values to troubleshoot the root cause. How can you achieve this via configuration?

</front>

---

<back>

Enable `store_failures` in the test configuration:

```yaml
data_tests:
  - not_null:
      config:
        store_failures: true
```

This causes dbt to create a table in the database containing the failing records for manual inspection.

</back>
