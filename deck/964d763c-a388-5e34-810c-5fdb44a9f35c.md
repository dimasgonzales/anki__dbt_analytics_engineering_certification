<front>

How does dbt determine configuration precedence between `dbt_project.yml` and the `config()` macro?

</front>

---

<back>

Configurations defined inside a model using the `{{ config() }}` macro take precedence over configurations defined in `dbt_project.yml`.

The hierarchy (from lowest to highest priority) is typically:
1. Default project config
2. `dbt_project.yml`
3. In-file `config()` macro

</back>
