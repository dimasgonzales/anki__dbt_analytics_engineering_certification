<front>

You want all models in the `marts/finance` directory to be materialized as `table`, except for `stg_payments` which should be a `view`. How do you configure this efficiently?

</front>

---

<back>

Define the directory-level default in `dbt_project.yml`, then override the specific model using the `config()` macro.

`dbt_project.yml`:
```yaml
models:
  my_project:
    marts:
      finance:
        +materialized: table
```

`stg_payments.sql`:
```jinja
{{ config(materialized='view') }}
```

</back>
