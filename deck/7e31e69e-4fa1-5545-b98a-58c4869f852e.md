<front>

Why does dbt store database credentials in `profiles.yml` rather than `dbt_project.yml`?

</front>

---

<back>

To prevent sensitive credentials from being checked into version control.

`profiles.yml` usually lives in the user's home directory (`~/.dbt/`) outside the git repository, while `dbt_project.yml` is part of the project source code.

</back>
