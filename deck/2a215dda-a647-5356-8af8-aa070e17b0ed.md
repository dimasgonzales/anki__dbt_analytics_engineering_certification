<front>

What is the effect of setting `contract: { enforced: true }` on a dbt model configuration?

</front>

---

<back>

It enforces hard guarantees on the model's schema. During `dbt run`, dbt verifies that the dataset produced matches the columns and data types defined in the YAML exactly. If there is a mismatch, the build fails.

</back>
