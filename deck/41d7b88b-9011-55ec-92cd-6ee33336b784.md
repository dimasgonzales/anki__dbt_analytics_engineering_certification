<front>

How do you explicitly define data types for columns in a dbt seed to override default type inference?

</front>

---

<back>

You must use the `+column_types` configuration in `dbt_project.yml` or a property YAML file. This allows you to map column names to specific SQL types (e.g., `varchar`, `date`) rather than relying on dbt's automatic inference.

Example:
```yaml
seeds:
  jaffle_shop:
    +column_types:
      zip_code: varchar(10)
```

</back>
