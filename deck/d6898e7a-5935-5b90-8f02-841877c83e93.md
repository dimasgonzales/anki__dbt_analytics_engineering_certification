<front>

When using `run_query` to execute DDL statements (like `GRANT` or `VACUUM`) inside a macro, why must you wrap the function call in a `{% do ... %}` tag?

</front>

---

<back>

The `{% do ... %}` tag evaluates the expression **without printing anything** to the rendered SQL output. 

Without it, the return value of `run_query` might be inadvertently written into the compiled SQL file, causing syntax errors.

</back>
