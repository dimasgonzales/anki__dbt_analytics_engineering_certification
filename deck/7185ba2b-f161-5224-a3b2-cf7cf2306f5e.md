<front>

Why is it recommended to use Python Virtual Environments (venv) or aliases when installing the dbt Cloud CLI on a machine with dbt Core?

</front>

---

<back>

Both dbt Core and the dbt Cloud CLI use the same binary command name (`dbt`). Installing them globally on the same machine causes path conflicts, leading to ambiguity regarding which version (local Core vs. remote Cloud) is executing your commands.

</back>
