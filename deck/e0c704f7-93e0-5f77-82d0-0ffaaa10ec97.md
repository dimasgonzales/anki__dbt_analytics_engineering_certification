<front>

You want to enable failure storage globally for your project, but disable it for one specific complex test to save storage. How would you structure this configuration?

</front>

---

<back>

Define the global default in `dbt_project.yml` and override it in the specific test's YAML block.

```yaml
# dbt_project.yml
data_tests:
  +store_failures: true

# properties.yml
  - unique:
      config:
        store_failures: false
```

</back>
