<front>

When defining column properties in YAML, what does setting `quote: true` accomplish?

</front>

---

<back>

It forces dbt to wrap the column name in quotes (e.g., `"order"` vs `order`) in the generated SQL.

This is necessary when a column name conflicts with a database reserved keyword (like `group`, `user`, or `order`).

</back>
