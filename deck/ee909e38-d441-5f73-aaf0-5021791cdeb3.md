<front>

You need to use a specific password or secret within the 'Extended Attributes' section for a connection configuration. How should you format this to ensure security?

</front>

---

<back>

Wrap the sensitive value in a Jinja `env_var` function. This prevents the plain-text secret from being exposed in logs or the UI configurations.

```yaml
password: '{{ env_var(''DBT_SECRET_PWD'') }}'
```

</back>
