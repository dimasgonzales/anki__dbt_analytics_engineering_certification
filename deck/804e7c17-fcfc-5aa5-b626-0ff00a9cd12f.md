<front>

How does setting `identifier: true` in a source configuration affect the output of the `{{ source() }}` function during compilation?

</front>

---

<back>

It wraps the table name (identifier) in **double quotes** in the compiled SQL.

For example, `{{ source('db', 'table') }}` compiles to `..."table"` instead of `...table`. This enforces case sensitivity on the database side.

</back>
