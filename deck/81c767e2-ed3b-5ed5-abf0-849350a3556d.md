<front>

How can you use the CLI to inspect the specific rows causing a data test failure without opening the compiled SQL in a separate tool?

</front>

---

<back>

Run the `dbt show` command targeting the specific test name. This executes the test query (which selects the failing records) and prints the results directly to the terminal.

```bash
dbt show --select "unique_customers_customer_id"
```

</back>
