<front>

What is the default behavior of the `info` argument in the dbt `log` function, and how do you change it to print to the CLI?

</front>

---

<back>

The default is `info=False`, which writes only to `dbt.log` (debug level). To print to standard output (CLI), you must explicitly set `info=True`.

```jinja
{{ log("Message", info=True) }}
```

</back>
