<front>

You want to materialize a specific seed file, `country_codes.csv`, into a custom schema named `reference`. You try adding `{{ config(schema='reference') }}` to the first line of the CSV but receive a parsing error. How should you fix this?

</front>

---

<back>

Remove the Jinja code from the CSV file. Instead, apply the configuration in your `dbt_project.yml` file or a dedicated properties YAML file.

Example in `dbt_project.yml`:
```yaml
seeds:
  my_project:
    country_codes:
      +schema: reference
```

</back>
