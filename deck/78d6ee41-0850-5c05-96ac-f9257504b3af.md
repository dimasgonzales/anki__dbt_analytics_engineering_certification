<front>

What is the primary performance benefit of using key-based distribution (`dist='column_name'`) in Redshift?

</front>

---

<back>

It achieves **data colocation**. By storing all rows with the same key value on the same compute node, dbt minimizes or eliminates the need for network I/O (shuffling/broadcasting) when joining tables on that specific column.

</back>
