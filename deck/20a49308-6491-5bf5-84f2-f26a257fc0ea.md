<front>

Your CI/CD pipeline generates documentation to verify lineage changes, but the `dbt docs generate` step is causing timeouts due to long query times on the information schema. How can you optimize this?

</front>

---

<back>

Update the pipeline command to use the `--empty-catalog` flag:
```bash
dbt docs generate --empty-catalog
```
This preserves the lineage visualization required for verification while removing the latency caused by querying database metadata.

</back>
