<front>

You are converting a Jinja-heavy SQL model to Python. How do you replicate the functionality of `{{ config(materialized='table') }}` and `{{ ref('orders') }}` inside the Python script?

</front>

---

<back>

You must use the `dbt` object methods instead of Jinja brackets. 

Use `dbt.config()` for configuration and `dbt.ref()` for referencing upstream models:
```python
def model(dbt, session):
    dbt.config(materialized='table')
    orders_df = dbt.ref('orders')
    return orders_df
```

</back>
