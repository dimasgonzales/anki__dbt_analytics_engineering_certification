<front>

In a dbt incremental model, what happens if you do not define a `unique_key` configuration?

</front>

---

<back>

The model functions as 'append-only'. dbt inserts all rows returned by the query into the destination table without checking for existing records, which creates duplicate rows if the data has been processed before.

</back>
