<front>

A macro call in your model fails with the error: `'status' is undefined`. The code is:
`{{ calculate_metric(status) }}`

What is the cause of this error?

</front>

---

<back>

The argument `status` is passed without quotes, so Jinja interprets it as a **variable** named `status`. Since no such variable was defined (via `{% set %}`), it errors. To pass the literal string 'status', you must quote it:
```sql
{{ calculate_metric('status') }}
```

</back>
