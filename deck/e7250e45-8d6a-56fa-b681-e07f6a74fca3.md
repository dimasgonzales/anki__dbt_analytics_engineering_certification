<front>

How can you ensure that specific warning configurations apply to all team members running a project without requiring them to type CLI flags?

</front>

---

<back>

You can define the configuration in the **`dbt_project.yml`** file under the `flags` key. This persists the settings project-wide.

Example:
```yaml
flags:
  warn_error_options:
    error: all
```

</back>
