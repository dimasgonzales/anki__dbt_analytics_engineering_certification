<front>

A model is failing with a database syntax error, but the Jinja code looks correct. How would you troubleshoot the exact SQL being sent to the warehouse?

</front>

---

<back>

Use the **Compile** feature (or `dbt compile` CLI command) and inspect the compiled SQL in the `target/` directory. This allows you to see the rendered SQL with all macros and variables resolved before it hits the database.

</back>
