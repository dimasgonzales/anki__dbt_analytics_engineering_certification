<front>

A custom macro is returning unexpected results. How can you inspect the value of a variable named `cutoff_date` during the compilation phase to troubleshoot?

</front>

---

<back>

Insert a `log` statement inside the macro using `info=True` to print the variable's value to the console during execution.

```jinja
{{ log("Current cutoff: " ~ cutoff_date, info=True) }}
```

</back>
