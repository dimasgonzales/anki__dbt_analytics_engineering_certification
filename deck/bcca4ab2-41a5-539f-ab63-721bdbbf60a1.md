<front>

Why does the ordinal position of columns in a `SELECT` statement matter for performance in certain file formats (like Delta Lake or Parquet)?

</front>

---

<back>

Some query engines only collect statistics for file pruning on the first N columns (often the first 32). Placing high-cardinality columns frequently used in filters (like IDs or Timestamps) at the beginning of the schema ensures the engine can effectively skip files during scans.

</back>
