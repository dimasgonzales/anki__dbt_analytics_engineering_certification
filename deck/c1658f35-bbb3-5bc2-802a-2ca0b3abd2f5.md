<front>

What is the primary purpose of defining Model Groups in `dbt_project.yml`?

</front>

---

<back>

Model Groups define ownership (such as team contact info) and create boundaries for `private` access control.

They allow you to segment a project so that `private` models can only be referenced by other models within the same group.

</back>
