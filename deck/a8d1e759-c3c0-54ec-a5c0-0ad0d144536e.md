<front>

You are versioning the `subscriptions` model. You want version 1 to use an existing file named `legacy_subs_logic.sql` instead of renaming it to match the dbt version convention. 

How do you configure this in YAML?

</front>

---

<back>

Use the `defined_in` property under the specific version:

```yaml
models:
  - name: subscriptions
    versions:
      - v: 1
        defined_in: legacy_subs_logic
```

</back>
