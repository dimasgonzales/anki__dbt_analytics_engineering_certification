<front>

A data test failed yesterday, creating a table with rows in the audit schema. Today, you fix the underlying data and run `dbt test` again (resulting in a pass). What happens to the data in the audit table?

</front>

---

<back>

The table is **replaced/emptied**. dbt recreates the table for that test on every run. If the test passes (0 failures), the table will effectively be empty, erasing the history of the previous failure.

</back>
