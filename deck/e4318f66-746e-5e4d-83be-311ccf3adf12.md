<front>

How do you dynamically configure a source database name in `sources.yml` based on the environment (e.g., dev vs. prod)?

</front>

---

<back>

Use the `env_var` Jinja function within the YAML file. This resolves the value at runtime based on the environment variables set in the system or dbt Cloud job context.

```yaml
database: "{{ env_var('SOURCE_DATABASE') }}"
```

</back>
