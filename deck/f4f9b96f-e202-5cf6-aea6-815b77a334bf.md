<front>

You triggered a production deployment via API immediately after a scheduled daily run started. You notice the API-triggered run is in a 'Queued' state rather than 'Running'. Why is this happening?

</front>

---

<back>

The job is queued to maintain **production integrity**. Unlike CI jobs which run in parallel, production jobs run sequentially to ensure that multiple processes do not attempt to overwrite tables or views in the production schema at the same time.

</back>
