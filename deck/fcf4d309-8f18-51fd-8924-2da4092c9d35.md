<front>

In dbt-postgres, what is the default incremental strategy when a `unique_key` is defined versus when it is not defined?

</front>

---

<back>

When a `unique_key` is defined, the default strategy is `delete+insert`. When no `unique_key` is defined, the default strategy is `append`. This differs from other adapters like Snowflake which typically default to `merge`.

</back>
