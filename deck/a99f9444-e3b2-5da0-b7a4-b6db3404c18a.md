<front>

Describe the "atomic swap" strategy dbt uses when materializing a table model.

</front>

---

<back>

To minimize downtime, dbt builds the new version of the model in a temporary table first. 

Once the build succeeds, dbt drops the old table and renames the temporary table to the production name (often within a single transaction on supported adapters).

</back>
