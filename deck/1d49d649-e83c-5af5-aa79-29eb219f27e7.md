<front>

How does dbt Cloud handle the `profiles.yml` connection configuration compared to dbt Core (CLI)?

</front>

---

<back>

dbt Cloud replaces the local `profiles.yml` file with **managed Environments** (e.g., Development, Deployment).

Connection details are configured in the dbt Cloud UI credentials settings rather than a file in the user's home directory, ensuring secure and consistent access control.

</back>
