<front>

Why is `dbt clone` useful for testing incremental models in a CI environment?

</front>

---

<back>

It allows you to simulate **incremental runs** without a full refresh. By cloning the production state of a model into the CI schema first, you establish a baseline that allows you to run the incremental logic on top, verifying that new data merges correctly.

</back>
