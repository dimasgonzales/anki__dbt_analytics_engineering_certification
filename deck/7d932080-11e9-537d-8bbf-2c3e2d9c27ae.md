<front>

Your CI pipeline is timing out because it runs `dbt run --full-refresh` on a massive incremental table. How would you refactor the pipeline steps to fix this?

</front>

---

<back>

Replace the full refresh with a clone-and-run strategy:

```bash
# 1. Clone prod state to CI schema
dbt clone --select big_incremental_model --state path/to/prod

# 2. Run incremental logic
dbt run --select big_incremental_model
```

</back>
