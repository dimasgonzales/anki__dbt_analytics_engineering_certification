<front>

Your model descriptions are becoming very long (50+ lines) and cluttering your `schema.yml` files, making the configuration hard to read. How should you refactor this project to improve maintainability?

</front>

---

<back>

Move the long descriptions into separate Markdown (`.md`) files using **docs blocks**.

1. Define the content in a `.md` file:
```markdown
{% docs my_long_desc %} ... {% enddocs %}
```
2. Reference it in the YAML:
```yaml
description: '{{ doc("my_long_desc") }}'
```

</back>
