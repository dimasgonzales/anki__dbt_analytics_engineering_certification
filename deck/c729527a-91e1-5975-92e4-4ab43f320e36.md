<front>

A complex model is failing with a generic SQL syntax error during `dbt run`, but the source code is full of Jinja macros making it hard to read. How would you troubleshoot this?

</front>

---

<back>

Go to the `target/compiled/{project_name}/models/...` directory and open the compiled SQL file for that model. Copy this raw SQL and run it directly in your data warehouse console to isolate the specific syntax error generated by the compiled logic.

</back>
