<front>

A developer hardcodes a table name `from "analytics"."prod"."stg_orders"` inside a model instead of using `ref('stg_orders')`. What is the major consequence regarding execution order?

</front>

---

<back>

dbt will **not** recognize the dependency. Consequently, dbt may attempt to run the downstream model *before* `stg_orders` is built or updated, potentially leading to errors or stale data.

</back>
