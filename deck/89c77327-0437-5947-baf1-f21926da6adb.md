A stakeholder asks where the logic for the `net_revenue` column resides. How would you find this efficiently without manually parsing SQL files?

</front>

---

<back>

Use **Column-Level Lineage** in dbt Explorer. Select the `net_revenue` column in the downstream model to visually trace its path back to upstream sources and identify exactly which transformations contribute to that specific field.

</back>
