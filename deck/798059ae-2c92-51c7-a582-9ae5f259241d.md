<front>

You need to apply `accepted_values` twice on a `status` column: once for current orders and once for archived orders. How do you avoid a naming collision error?

</front>

---

<back>

You must explicitly define a unique `name` for at least one (or both) of the tests.

```yaml
- accepted_values:
    name: check_current_status
    arguments: { values: ['placed'] }
    config: { where: "order_date = current_date" }
```

</back>
