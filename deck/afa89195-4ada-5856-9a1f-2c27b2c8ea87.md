<front>

You need to snapshot a table called `shipping_status` that lacks a reliable updated timestamp, but you need to track changes to the `status` and `delivery_date` columns. How would you configure this in dbt?

</front>

---

<back>

You must use the **check** strategy. In the configuration, specify the strategy and list the columns to monitor in `check_cols`.

```yaml
config(
  target_schema='snapshots',
  unique_key='shipment_id',
  strategy='check',
  check_cols=['status', 'delivery_date']
)
```

</back>
