<front>

You want to prevent your pipeline from running transformations if the upstream raw data hasn't been updated in the last 24 hours. How would you achieve this workflow?

</front>

---

<back>

First, define a `freshness` block in your source YAML. Then, configure your orchestrator to run:

```bash
dbt source freshness
```

If this command fails (returns a non-zero exit code), stop the job before executing `dbt build`.

</back>
