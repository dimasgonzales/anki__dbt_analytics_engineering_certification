<front>

You have changed a column's data type in the SQL logic of an existing incremental model, and the next run fails with a database type mismatch error. How do you resolve this?

</front>

---

<back>

Run the model with the `--full-refresh` flag to drop the old table and rebuild it with the new schema:
```bash
dbt run --select my_model --full-refresh
```

</back>
