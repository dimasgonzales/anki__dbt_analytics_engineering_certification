<front>

What is the syntactic difference between configuring `partition_by` and `cluster_by` for BigQuery models in dbt?

</front>

---

<back>

`partition_by` requires a dictionary defining the `field`, `data_type`, and `granularity`. `cluster_by` requires a list (or string) of column names to sort by.

Example:
```python
config(
  partition_by={'field': 'created_at', 'data_type': 'timestamp'},
  cluster_by=['user_id', 'status']
)
```

</back>
