<front>

Model A uses `run_query()` to fetch logic dynamically. Model B references Model A (`ref('model_a')`). When running `dbt compile`, why is Model B not fully analyzed in the initial pass?

</front>

---

<back>

Because Model A forces JIT rendering via introspection, dbt must also delay the analysis of Model B until runtime. Model B cannot be validated until Model A has theoretically run and the database state is known.

</back>
