<front>

What Jinja function allows you to inject values from the operating system into `dbt_project.yml` configurations?

</front>

---

<back>

`env_var`.

It is used to dynamically set configurations like target databases, schema names, or test severities based on the environment (e.g., Local vs. CI).

Example:
```yaml
key: "{{ env_var('VARIABLE_NAME', 'default_value') }}"
```

</back>
