<front>

What is the specific syntax requirement for defining configuration keys (like `delimiter` or `enabled`) inside `dbt_project.yml` versus a properties file `config` block?

</front>

---

<back>

In `dbt_project.yml`, configuration keys must be prefixed with a `+` (e.g., `+delimiter`) to distinguish them from directory names. In a properties file `config` block, the `+` prefix is **not** used.

```yaml
# properties.yml config block
config:
  delimiter: "|"
```

</back>
