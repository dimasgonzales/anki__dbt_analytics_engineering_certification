<front>

What happens by default when you run `dbt seed` against an existing table, and what command flag is required if the CSV columns have changed?

</front>

---

<back>

By default, dbt performs a TRUNCATE and INSERT to update data while preserving the table object. If columns change, you must use `--full-refresh` to force a DROP and CREATE.

</back>
