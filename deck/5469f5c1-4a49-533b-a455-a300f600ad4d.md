<front>

In dbt Core's Semantic Versioning (Major.Minor.Patch), which update type introduces breaking changes requiring code refactoring?

</front>

---

<back>

A **Major** version update (e.g., v1.x to v2.x) introduces breaking changes. 

Minor versions (e.g., v1.8 to v1.9) add features but are backwards compatible, while Patch versions (e.g., v1.9.0 to v1.9.1) contain only bug or security fixes.

</back>
