<front>

How do you configure commands in a dbt Cloud job to only operate on modified nodes in the DAG? (e.g. for a slim CI job)

</front>

---

<back>

Using the `-s state:modified` argument. If you add a "+" at the end, the job will also operate on any downstream nodes in the dag (recommended for CI purposes to ensure changes didn't break downstream dependent models). For example: `dbt run -s state:modified+`

</back>
