<front>

Your team wants the standard `unique` test to automatically ignore null values project-wide. How should you implement this?

</front>

---

<back>

Define a custom generic test named `unique` in your project's `tests/generic/` directory that includes logic to filter out nulls.

```sql
{% test unique(model, column_name) %}
  select *
  from {{ model }}
  where {{ column_name }} is not null
  -- rest of uniqueness logic
{% endtest %}
```

</back>
