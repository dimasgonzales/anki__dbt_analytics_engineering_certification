<front>

A developer writes the following incremental filter for a BigQuery model: `where event_date >= {{ _dbt_max_partition }}`. Why does this fail?

</front>

---

<back>

It fails because `_dbt_max_partition` is a BigQuery scripting variable, not a Jinja variable. It should be referenced directly in the SQL without curly braces.

Correct usage:
```sql
where event_date >= _dbt_max_partition
```

</back>
