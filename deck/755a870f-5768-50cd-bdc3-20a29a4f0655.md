<front>

A developer writes a macro `get_limit()` using the syntax `{{ return(100) }}`. When compiling the model, they notice unexpected artifacts in the SQL or Jinja rendering issues. How should the macro syntax be corrected?

</front>

---

<back>

Change the syntax to use the **do** tag:

```jinja
{% do return(100) %}
```

This executes the return statement as a logic command rather than an output expression.

</back>
