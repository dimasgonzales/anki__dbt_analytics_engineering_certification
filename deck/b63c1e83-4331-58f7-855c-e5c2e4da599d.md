<front>

What is the primary purpose of the `filter` configuration property within a dbt source freshness block?

</front>

---

<back>

The `filter` property optimizes performance and reduces cost by restricting the SQL query used to check freshness. It applies a `WHERE` clause (or equivalent) to limit the scan to a recent subset of data rather than scanning the entire table for the maximum timestamp.

</back>
