<front>

In a dbt `sources.yml` file, if quoting preferences are defined at both the top `source` level and the specific `table` level, which configuration applies?

</front>

---

<back>

The **table-level** configuration takes precedence. 

dbt uses a hierarchy where more specific configurations override general ones. If a source defines `identifier: true` but a specific table within that source defines `identifier: false`, the table will **not** be quoted.

</back>
