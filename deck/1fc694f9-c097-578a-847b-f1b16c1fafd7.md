<front>

What happens when you modify columns in a seed CSV file and run `dbt seed` without any flags?

</front>

---

<back>

The command will likely fail. By default, dbt attempts to truncate the existing table and insert the data. Since the table structure in the warehouse does not match the new CSV structure, the database will reject the insert operation.

</back>
