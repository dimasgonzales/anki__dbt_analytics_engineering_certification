<front>

If a data validation requirement isn't covered by the four native dbt tests, what is the recommended first step before writing a custom test?

</front>

---

<back>

Check if the test exists in the `dbt-utils` package. This community package contains many common tests (like `unique_combination_of_columns` or `recency`) that extend dbt's native capabilities.

</back>
