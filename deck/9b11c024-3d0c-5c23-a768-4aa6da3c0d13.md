<front>

Which Jinja context variable is commonly used to apply logic (such as data limits) only in specific environments like 'dev' or 'prod'?

</front>

---

<back>

`target.name`

This variable contains the name of the target specified in your `profiles.yml` file (e.g., 'dev', 'prod', 'ci'), allowing you to conditionally compile SQL based on where the code is running.

</back>
