<front>

You need to share a set of utility macros across multiple teams. Should you define this as a 'package' or 'project' dependency?

</front>

---

<back>

You should use a **package** dependency. Packages are designed for sharing logic libraries (macros) to be installed and parsed locally. Project dependencies are specifically for referencing data models (datasets) across projects via dbt Mesh.

</back>
