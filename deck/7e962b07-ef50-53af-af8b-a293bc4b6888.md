<front>

According to dbt best practices, what is the specific purpose of inline SQL comments (e.g., `--` or `/* */`) versus YAML descriptions?

</front>

---

<back>

Inline SQL comments are for developers/maintainers to explain complex code logic, implementation details, or mark TODOs. YAML descriptions are for data consumers to understand business context and definitions via the generated documentation site.

</back>
