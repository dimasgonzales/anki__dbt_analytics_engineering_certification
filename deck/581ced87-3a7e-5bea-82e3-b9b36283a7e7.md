<front>

How do you map a messy physical table name (e.g., `RAW_DATA_2023`) to a clean logical name (e.g., `orders`) in a dbt source configuration?

</front>

---

<back>

Use the `identifier` property in your YAML configuration. Set the `name` property to your desired logical name (`orders`) and the `identifier` property to the physical table name (`RAW_DATA_2023`).

```yaml
tables:
  - name: orders
    identifier: RAW_DATA_2023
```

</back>
