<front>

How does the `ephemeral` materialization differ from `table` and `view` regarding database object creation?

</front>

---

<back>

`ephemeral` models do **not** create any objects (tables or views) in the database.

Instead, dbt interpolates the code from the ephemeral model as a Common Table Expression (CTE) directly into the SQL of downstream models.

</back>
