Your CI/CD pipeline runs `dbt docs generate`, but the resulting documentation site does not show the most recent table statistics or column changes from the warehouse. What is the likely troubleshooting step?

</front>

---

<back>

Ensure that `dbt docs generate` successfully created a new `catalog.json`. The documentation relies on this specific artifact to synchronize metadata from the warehouse (like column types and comments) with the definition in `manifest.json`.

</back>
