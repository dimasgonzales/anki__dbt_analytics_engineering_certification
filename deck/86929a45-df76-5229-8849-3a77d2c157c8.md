<front>

You have an analysis file `analyses/kpi_export.sql` that uses `{{ ref('orders') }}`. You need to copy the raw SQL to run in a separate BI tool. How do you get the SQL with the correct schema and table names?

</front>

---

<back>

Execute `dbt compile --select kpi_export`. Then, navigate to the `target/compiled/...` directory to open the generated file. You cannot simply copy the source code because the BI tool will not understand the `{{ ref() }}` syntax.

</back>
