<front>

Why is the `dbt` templater required when using SQLFluff with a dbt project?

</front>

---

<back>

Standard SQL linters cannot interpret Jinja syntax like `{{ ref() }}` or `{{ config() }}`. The `dbt` templater compiles the Jinja code into valid data warehouse SQL before the linter analyzes it.

Configure it in `.sqlfluff`:
```ini
[sqlfluff]
templater = dbt
```

</back>
