<front>

What does the `run_query` macro do in dbt?

</front>

---

<back>

`run_query` runs a SQL statement against your data warehouse immediately and returns the results as an Agate table object to the Jinja context.

This allows you to inspect data values programmatically during the run to generate dynamic SQL logic.

</back>
