<front>

What are the top-level keys that must be defined in `dbt_project.yml` for a project to be valid?

</front>

---

<back>

A valid `dbt_project.yml` requires:

1.  **name**: The project name (must be `snake_case`).
2.  **profile**: The name of the connection profile used in `profiles.yml`.

> **Note:** The `version` and `config-version: 2` keys are now optional starting in dbt Core v1.5. Historical dbt Core versions prior to v1.5 required `config-version`



</back>
