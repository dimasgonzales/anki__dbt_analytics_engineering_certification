<front>

How do you apply warehouse-specific physical table properties, such as Snowflake clustering keys or Databricks Z-Ordering, in dbt?

</front>

---

<back>

You apply these properties using the `{{ config(...) }}` block within the model file or in `dbt_project.yml`. This ensures infrastructure optimization is treated as code and version-controlled.
```sql
{{ config(cluster_by=['user_id']) }}
```

</back>
