<front>

What is 'Slim CI' in dbt and which selector is primarily used to achieve it?

</front>

---

<back>

Slim CI is a practice where dbt only runs models that have changed and their downstream dependencies, rather than the full project. This is achieved using the `state:modified` selector combined with a reference manifest.

</back>
