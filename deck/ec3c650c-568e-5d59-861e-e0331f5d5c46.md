<front>

How does the 'Compile' function in dbt differ from a simple Jinja render regarding context variables?

</front>

---

<back>

The 'Compile' function resolves SQL using the model's specific **execution context**. This means variables like `{{ this }}`, `{{ is_incremental() }}`, and adapter-specific configurations are resolved based on the active config, distinguishing it from a generic string render.

</back>
