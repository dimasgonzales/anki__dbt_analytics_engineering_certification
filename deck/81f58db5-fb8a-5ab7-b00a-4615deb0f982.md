<front>

What specific configuration parameter is required when using the `check` strategy in dbt snapshots, in addition to `unique_key` and `strategy`?

</front>

---

<back>

The `check` strategy requires the `check_cols` parameter. This parameter accepts either a list of specific column names to monitor for changes or the string `'all'` to check every column in the row.
```yaml
config:
  strategy: check
  check_cols: ['status', 'amount'] # or 'all'
```

</back>
