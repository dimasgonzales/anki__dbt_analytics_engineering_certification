<front>

What test configuration is recommended for Materialized Views to allow for the inspection of failing records after the test run completes?

</front>

---

<back>

`store_failures_as = 'view'`

Configuring this (available in dbt v1.7+) saves the rows that failed the test into a persistent database view, allowing for easier debugging of continuous pipelines where data changes rapidly.

</back>
