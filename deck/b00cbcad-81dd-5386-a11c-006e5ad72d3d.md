<front>

What does the `model` variable represent within the dbt Jinja context?

</front>

---

<back>

The `model` variable is a graph object representing the node (model, seed, or snapshot) currently being executed. It provides access to metadata such as configuration settings (`model.config`), the name (`model.name`), and schema information, allowing for dynamic logic in macros and hooks.

</back>
