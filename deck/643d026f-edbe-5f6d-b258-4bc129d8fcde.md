<front>

When defining configurations for a dbt model, does the `{{ config() }}` block inside the SQL file or the `dbt_project.yml` file take precedence?

</front>

---

<back>

The in-model **`{{ config() }}` block takes precedence**. Configurations defined directly within the model file override any conflicting configurations set at the folder or project level in `dbt_project.yml`.

</back>
