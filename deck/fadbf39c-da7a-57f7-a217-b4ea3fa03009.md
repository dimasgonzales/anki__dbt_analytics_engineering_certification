<front>

You are reviewing a PR with the following snapshot configuration. What will happen when `dbt compile` is run?

```sql
{% snapshot orders_scd %}
    {{ config(
      target_schema='snapshots',
      unique_key='id',
      strategy='timestamp',
      updated_at='updated_at',
      materialized='table'
    ) }}
    select * from {{ source('raw', 'orders') }}
{% endsnapshot %}
```

</front>

---

<back>

The compilation will fail with a **Parsing Error**.

You cannot configure `materialized='table'` for a snapshot. To fix this, remove the `materialized='table'` line; dbt automatically handles the materialization strategy for blocks defined within `{% snapshot %}` tags.

</back>
