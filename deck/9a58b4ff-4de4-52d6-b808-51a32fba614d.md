<front>

What are the four built-in generic data tests available in dbt?

</front>

---

<back>

1. `unique`: Verifies that every value in a column is unique.
2. `not_null`: Verifies that a column contains no null values.
3. `accepted_values`: Verifies that a column only contains values from a provided list.
4. `relationships`: Verifies referential integrity (foreign keys) between two models.

</back>
