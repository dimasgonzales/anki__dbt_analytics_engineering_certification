<front>

How do you apply built-in tests (like `unique` or `not_null`) to a Python model defined in a `.py` file?

</front>

---

<back>

You define them in a **YAML file** (e.g., `schema.yml`) exactly as you would for SQL models. dbt treats the Python model's output table as a standard relation and runs SQL-based tests against it after materialization.

</back>
