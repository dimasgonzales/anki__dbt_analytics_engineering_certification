<front>

Why is the `unique_key` configuration mandatory for dbt Snapshots?

</front>

---

<back>

Snapshots implement SCD Type 2 logic, which requires tracking changes to a specific entity over time. The `unique_key` is essentially the primary key used to match incoming source records to existing history to determine if a row has changed, is new, or remains the same.

</back>
