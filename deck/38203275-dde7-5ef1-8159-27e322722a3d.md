<front>

What is the specific behavior of the `is_incremental()` macro when the command `dbt run --full-refresh` is executed?

</front>

---

<back>

The `is_incremental()` macro returns `false`, regardless of whether the table exists or is configured as incremental. This bypasses the incremental filter logic, drops the existing table, and rebuilds it entirely from scratch.

</back>
