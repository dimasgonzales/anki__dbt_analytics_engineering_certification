<front>

What is the difference in behavior for breaking changes in an enforced contract between a **versioned** model and a **non-versioned** model?

</front>

---

<back>

For a **non-versioned** model, breaking changes raise a **WARNING**.

For a **versioned** model, breaking changes raise an **ERROR** and fail the build.

</back>
