<front>

A consumer double-clicks a node in the dbt Explorer lineage graph representing a public model from another project, but the resource-level lineage view does not open. What is the likely cause?

</front>

---

<back>

The user lacks permissions for the upstream project. Without explicit access to the source project, users are restricted to viewing public details only and cannot inspect the internal lineage or private dependencies of that project.

</back>
