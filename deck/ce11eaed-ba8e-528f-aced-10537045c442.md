<front>

What happens by default if a privilege is defined in `dbt_project.yml` and then redefined for that same privilege in a specific model?

</front>

---

<back>

The specific model configuration **replaces (clobbers)** the project-level configuration entirely. It does not merge the lists unless special syntax is used.

</back>
