<front>

How does dbt handle configuration precedence between `dbt_project.yml` and the in-file `{{ config() }}` macro?

</front>

---

<back>

The in-file `{{ config() }}` macro always takes precedence over configurations defined in `dbt_project.yml`. This allows you to set global defaults at the project or directory level while overriding specific settings for individual models.

</back>
