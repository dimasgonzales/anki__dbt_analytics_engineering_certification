<front>

A developer notices that the database schema created during a dbt Cloud CI job has disappeared after their Pull Request was merged. What is the likely cause?

</front>

---

<back>

This is standard dbt Cloud behavior. 

When a Pull Request is closed or merged, dbt Cloud automatically drops the temporary schema associated with that PR to keep the data warehouse clean and minimize storage costs.

</back>
