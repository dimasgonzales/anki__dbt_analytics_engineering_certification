<front>

You have a folder `models/marts` configured as `+materialized: table` in `dbt_project.yml`. Inside that folder, you have a model `daily_sales.sql` containing `{{ config(materialized='view') }}`.

How will dbt materialize `daily_sales`?

</front>

---

<back>

It will be materialized as a **view**.

The in-file configuration `{{ config(materialized='view') }}` overrides the directory-level configuration set in `dbt_project.yml`.

</back>
