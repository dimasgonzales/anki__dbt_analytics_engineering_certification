<front>

You write a dbt model that `JOIN`s a massive table first, followed by a small lookup table. When viewing the `EXPLAIN` plan, you see the database is executing the join on the small table first. Why?

</front>

---

<back>

The database optimizer analyzed the query and generated a **Physical Plan** that reordered the operations. It determined that filtering or joining the small table first reduces the memory usage and row processing required before interacting with the massive table.

</back>
