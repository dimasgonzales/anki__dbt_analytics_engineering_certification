<front>

What is the required hierarchy structure when configuring specific seeds in `dbt_project.yml`?

</front>

---

<back>

The required hierarchy is `seeds` -> `[project_name]` -> `[seed_name]`. Omitting the project name is a common configuration error that results in the config not being applied.

</back>
