<front>

How can you filter the dataset used by a generic test without modifying the SQL of the model itself?

</front>

---

<back>

Use the `where` configuration within the test's `config` block. This applies a SQL filter to the data before the test logic runs.

```yaml
data_tests:
  - unique:
      config:
        where: "created_at > '2023-01-01'"
```

</back>
