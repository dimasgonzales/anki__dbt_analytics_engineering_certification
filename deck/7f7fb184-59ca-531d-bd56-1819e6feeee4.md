<front>

What is the primary use case for the `audit_helper` package during a migration project?

</front>

---

<back>

It is used to programmatically audit and compare data between a legacy SQL table and a new dbt model.

It provides macros (like `compare_relations`) to identify row-by-row discrepancies to ensure the refactored logic matches the original.

</back>
