<front>

A model run is taking unusually long and the logs show the number of output rows is exponentially higher than the input rows. What is the likely cause and solution?

</front>

---

<back>

The cause is likely an **exploding join** (Cartesian product) where the join keys are not unique. To fix this, inspect the join logic to ensure the grain matches or pre-aggregate the joined table (e.g., using `GROUP BY`) to ensure uniqueness before joining.

</back>
