<front>

Does the built-in `unique` test support checking uniqueness across multiple columns (composite keys) by default?

</front>

---

<back>

No. The built-in `unique` test targets a **single column**. To test composite keys, you must either use a surrogate key, write an expression in the test configuration, or use a community package test like `dbt_utils.unique_combination_of_columns`.

</back>
