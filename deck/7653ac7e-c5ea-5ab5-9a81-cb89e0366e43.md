<front>

A developer pushes a 600MB CSV export of historical transaction logs to the `seeds/` folder, causing the CI/CD pipeline to time out during execution. What is the correct architectural fix?

</front>

---

<back>

Remove the file from the dbt project and load the data into the warehouse using a dedicated EL (Extract-Load) tool or cloud storage loader. Once loaded, configure the table as a **Source** in dbt using a `.yml` file so models can reference it using `{{ source('schema', 'table') }}`.

</back>
