<front>

How do you execute database maintenance commands like `VACUUM` or `ANALYZE TABLE` that do not return data within a dbt run?

</front>

---

<back>

Use `post-hooks` (or `on-run-end` hooks) configured in the model or project file. These hooks allow executing raw SQL statements for administrative tasks after a model is built, as standard dbt models are designed for `SELECT` statements.

</back>
