<front>

Your BI team reports that a dashboard connected to a dbt model is timing out when loading, even though the dbt run completes successfully in a few minutes. The model is currently a **view**. How should you resolve this?

</front>

---

<back>

Change the materialization to a **table**.

The issue is query latency (reading data), not build latency (transforming data). Materializing as a table pre-computes the logic, making the dashboard queries significantly faster.

</back>
