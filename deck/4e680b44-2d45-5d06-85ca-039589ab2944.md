<front>

A complex test is failing, but the error message doesn't explain why specific rows are invalid. How would you troubleshoot this efficiently?

</front>

---

<back>

1. Inspect the compiled SQL code in the `target/compiled` directory to ensure the logic is correct.
2. Run `dbt test --store-failures`. This creates a table containing the specific rows causing the failure, allowing you to inspect the data directly using SQL.

</back>
