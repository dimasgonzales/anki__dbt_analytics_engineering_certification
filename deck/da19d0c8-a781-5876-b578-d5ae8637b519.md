<front>

You need to assert that the combination of `transaction_date` and `store_id` is unique in a table. How would you accomplish this since the native `unique` test only accepts one column?

</front>

---

<back>

Use the `unique_combination_of_columns` test from the `dbt-utils` package.

Example:
```yaml
tests:
  - dbt_utils.unique_combination_of_columns:
      combination_of_columns:
        - transaction_date
        - store_id
```

</back>
