<front>

Your CI pipeline currently rebuilds the entire warehouse on every commit, causing long wait times. How can you optimize this to only build changed models while still referencing existing production data for upstream dependencies?

</front>

---

<back>

Implement **Slim CI** by configuring the job to use state comparison. Pass the `--defer` flag to reference the production environment for unchanged upstream models, and use `--select state:modified+` to build only the new code and its downstream dependencies.

</back>
